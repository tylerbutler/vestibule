# Product Requirements Documents

This folder contains product requirements documents (PRDs) for vestibule.

Each PRD describes a feature, capability, or change to the project. PRDs are used to capture requirements, scope, and design decisions before implementation begins.

## Contents

Place PRD files here as Markdown documents. A typical PRD includes:

- **Problem statement** - What problem does this solve?
- **Goals and non-goals** - What's in scope and out of scope?
- **Requirements** - Specific functional and non-functional requirements
- **Design** - High-level approach or architecture
- **Open questions** - Unresolved decisions or areas needing investigation
